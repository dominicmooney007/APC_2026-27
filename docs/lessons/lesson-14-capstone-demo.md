# Lesson 14 — Mini-Capstone: Demo Day & Reflection

> Finish it, show it working, and take stock of a semester across three platforms.

**Platform:** Any
**Estimated time:** ~120 min hands-on
**Prerequisites:** [Lesson 13](lesson-13-capstone-build.md) — proposal signed, build underway

## Today's plan (~120 min)

| Time | Phase | What you're doing |
|---|---|---|
| ~60 min | Finish | Final build time — success criteria first, polish last |
| ~40 min | Demo day | Everyone presents, everyone watches |
| ~20 min | Reflect | Written reflection + repo finalized |

## Finishing (~60 min)

Priority order when time is short:

1. **Meet your success criteria** — the ones in your signed proposal. That's what's graded.
2. **Survive a cold start** — unplug, replug, run. Demos die at this step; rehearse it once.
3. **README complete** — goal, wiring table, how to run, known issues. "Known issues" honestly listed reads as engineering maturity, not failure.
4. Polish, if and only if 1–3 are done.

## Demo day (~40 min)

Three minutes each, at your bench:

1. **What it does** — one sentence, then *show it working*.
2. **How it works** — trace the path: sensor → decision → output. Point at the wire, point at the line of code.
3. **The hard part** — the problem that cost you most, and how you got past it (or didn't — honest post-mortems score fine).

While watching others: write down **one thing you'd steal** from someone else's project. It goes in your reflection.

!!! tip "If the demo breaks live"
    Narrate what *should* happen, show the code path, and debug calmly for a minute — grace under a broken demo is itself a skill being noticed. Your README and repo prove the work exists.

## Reflection (~20 min, submitted)

Short answers, in your project README or the hand-in doc:

1. You used three platforms this semester — **Arduino, Pi, Whisplay**. For your capstone you picked one. Why that one, and when would each of the others have been the right call?
2. What's one bug from this semester you'll never make again — and what's the *general rule* it taught you?
3. The thing you'd steal from someone else's demo — what and why?
4. If you had three more weeks, what would version 2 do?
5. Semester 2 pitch: one thing you want to learn or build next.

## What "done" looks like

- [ ] Demo delivered
- [ ] Repo folder: code + complete README + reflection
- [ ] Bench cleared, kit sorted, boards shut down properly — one last time

That's Semester 1: from a bare LED on a breadboard to a self-scoped device on the platform of your choice. Semester 2 builds from here.
